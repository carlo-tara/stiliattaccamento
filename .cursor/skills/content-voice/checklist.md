# Checklist supervisione contenuti

Usa questa checklist per ogni pagina o cluster **dopo** la bozza e **prima** del report finale.

## Scala di gravità

| Livello | Significato |
|---------|-------------|
| 🔴 Bloccante | Da correggere obbligatoriamente |
| 🟡 Importante | Da correggere nel cluster |
| 🟢 Suggerimento | Miglioramento consigliato |

---

## 1. Tono e voce

- [ ] Suona da compagno di viaggio informato / amico esperto
- [ ] Prevalente **tu**; **noi** solo dove include naturalmente
- [ ] Empatia concreta, non generica ("Comprendiamo le tue difficoltà" ❌)
- [ ] Non giudicante — direttezza solo dove serve e senza etichettare la persona
- [ ] Mix naturale di frasi brevi e lunghe
- [ ] Informalità leggera ok ("beh", pause) — senza esagerare
- [ ] Modulazione coerente col tipo di pagina (vedi SKILL.md)
- [ ] Nessun salto brusco teoria ↔ pratica nella stessa sezione
- [ ] 🔴 Nessun tono da manuale clinico freddo o da brochure wellness

## 2. Filosofia e messaggio

- [ ] **Consapevolezza, non guarigione** — nessuna eccezione
- [ ] Verbi: riconoscere, osservare, integrare (non guarire/curare/fixare)
- [ ] EFT: ciclo vs persona, bisogni legittimi, A.R.E. dove pertinente
- [ ] 🔴 Nessuna promessa di "diventare Secure" o "guarire"
- [ ] Speranza sobria: cambiamento = riconoscimento pattern, non perfezione

## 3. Terminologia

- [ ] **Oscillante** (mai disorganizzato nel testo visibile)
- [ ] Linguaggio inclusivo: persona, partner, chi
- [ ] 🔴 Nessun pattern vietato da `style-validator.js`
- [ ] 🔴 Nessun pattern da [reference.md](reference.md#anti-pattern-ia-italiano)
- [ ] Termini tecnici spiegati alla prima occorrenza
- [ ] Tooltip `<abbr class="wiki-term">` dove utile
- [ ] Inglese tecnico (body scan, focusing…) mantenuto + spiegato
- [ ] Gerarchia teorica rispettata: EFT → Jung → sistemica → Process Work

## 4. Bilanciamento 70/30

- [ ] ~70% contenuto pratico (cosa fare/notare/dire)
- [ ] ~30% teoria semplificata
- [ ] Semplificazioni dichiarate quando necessario
- [ ] Archetipi/metafore accompagnati da sguardo pratico

## 5. Struttura e formattazione

- [ ] Header: maiuscole naturali (uniformati, no ANSIOSO ALTO)
- [ ] Elenchi con struttura **variata** (non tutti uguali)
- [ ] Grassetto funzionale, non decorativo
- [ ] Emoji solo ⚠️ per avvisi clinici
- [ ] Storie: terza persona + dialoghi
- [ ] 🟡 Corrette inconsistenze Title Case / MAIUSCOLO nel cluster

## 6. Accuratezza concettuale

- [ ] Attaccamento come pattern, non identità fissa
- [ ] Modello a gradienti coerente (ansia/evitamento, livelli)
- [ ] Claim scientifici supportabili
- [ ] 🟡 Fonti citate dove possibile
- [ ] 🟢 Bibliografia aggiunta dove manca
- [ ] 🔴 Nessun contenuto che sostituisce terapia/diagnosi

## 7. Disclaimer clinici

- [ ] Presenti su profili Alto, oscillante, trauma, crisi
- [ ] Tono vicino (compagno di viaggio), non allarmistico
- [ ] Non ripetitivi eccessivamente nella stessa pagina
- [ ] Link a `quando-cercare-aiuto.html` dove appropriato

## 8. Coerenza cluster

- [ ] Stessi termini per stessi concetti tra pagine correlate
- [ ] Tono allineato tra pagine dello stesso cluster
- [ ] Intensità basso/medio/alto coerente
- [ ] Cross-link aggiornati

## 9. SEO e GEO

- [ ] Title e meta description chiari e unici
- [ ] H1 unico, gerarchia heading corretta
- [ ] Primo paragrafo risponde all'intento della pagina
- [ ] Schema.org coerente col contenuto
- [ ] Definizioni esplicite per citabilità (AI search)

## 10. Accessibilità e leggibilità

- [ ] Alt text descrittivi sulle immagini
- [ ] Link con testo significativo
- [ ] Paragrafi e liste scansionabili
- [ ] Nessuna informazione solo tramite colore

## 11. Personas

- [ ] Chiara: validazione + chiarezza ✅ priorità
- [ ] Marco: step pratici presenti
- [ ] Luca: tono non peggiorativo
- [ ] Sofia: nessuna colpevolizzazione del partner
- [ ] Elena/Andrea: struttura wiki + fonti

## 12. Validazione tecnica

- [ ] `npm run test:validation` passa (dopo modifica HTML)
- [ ] HTML semantico preservato
- [ ] Nessun trailing whitespace / encoding UTF-8

## Post-check

- [ ] Testo riscritto consegnato
- [ ] Report con issue per gravità
- [ ] Tagli proposti documentati se applicabili
